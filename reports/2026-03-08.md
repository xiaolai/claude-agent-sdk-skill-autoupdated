## Errors

### Typecheck — Exit Code 1 (False Positive)

The import-resolution check script constructs a combined `from ... import (...)` statement by joining export names. Today's run produced a leading comma, generating invalid syntax:

```
  File "<string>", line 1
    from claude_agent_sdk import (,AgentDefinition,ClaudeAgentOptions,...
                                  ^
SyntaxError: invalid syntax
  FAIL: Some template imports don't resolve
  FAIL:   Cannot import: (
```

**Likely cause:** A script-side bug in how the import list is assembled (an empty string or `None` entry was prepended before the first real name, producing `,AgentDefinition,...`). The actual SDK imports in the templates are valid — all 13 other syntax checks passed. The `(` character appearing as a "failed import name" confirms this is purely a script artifact.

**Impact:** Non-blocking. Pipeline `outcomes.typecheck` is recorded as `"success"`. No template changes required.

**Suggested fix:** Inspect the import-check script's list-join logic and filter empty/null entries before construction.
