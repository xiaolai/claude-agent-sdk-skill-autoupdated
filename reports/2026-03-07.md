```
  FAIL:   Cannot import: (

=========================================

  Failed: 2
=========================================

```

**Likely cause**: The typecheck script is generating an import string with a leading comma after the opening parenthesis — a bug in the script itself, not in any template. This is the same class of false positive seen on 2026-03-06.

**Impact**: None — verify step passed 28/28 checks independently, confirming templates are valid. Pipeline outcome was correctly marked as success.

**Suggested fix**: Patch the typecheck script to strip leading/trailing commas when constructing the import string for resolution checks.
