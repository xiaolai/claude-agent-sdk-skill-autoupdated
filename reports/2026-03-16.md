### Typecheck — exit code 1 (false-positive, non-blocking)

```
=== Python import resolution check ===
  File "<string>", line 1
    from claude_agent_sdk import (,AgentDefinition,ClaudeAgentOptions,...
                                  ^
SyntaxError: invalid syntax
  FAIL: Some template imports don't resolve
  FAIL:   Cannot import: (

=========================================
  Passed: 13
  Failed: 2
=========================================

TEMPLATE CHECK FAILED — 2 issue(s)
```

**Likely cause**: The typecheck script constructs a Python import string by joining symbols with commas, producing `(,AgentDefinition,...` with a leading comma — invalid Python syntax. The script is checking its own generated string, not actual SDK code.

**Suggested fix**: Strip leading/trailing commas when building the test import string in the typecheck script.

**Impact**: None — pipeline outcomes records `typecheck: "success"`. This false-positive has appeared in multiple prior runs.
