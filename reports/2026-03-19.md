### Python

Updates in `SKILL-python.md`:
- **KI #28 added** — `DEBUG` environment variable corrupts JSON protocol in Docker/Linux. When `DEBUG` is set to any non-empty value, the bundled CLI's sandbox layer writes `[SandboxDebug]` lines to stdout; the SDK's transport tries to parse them as JSON, causing all message parsing to fail (silent empty results or `Control request timeout: initialize`). Workaround: pass `env={**os.environ, "DEBUG": ""}` in `ClaudeAgentOptions`. Source: issue #347.
- **KI #23 updated** — added reference to PR #699 which proposes a `--thinking-disabled` flag fix for the `thinking={"type":"disabled"}` → `--max-thinking-tokens 0` conversion bug; not yet released.
- **Last verified** date updated to 2026-03-19.

## Errors

### Typecheck — false-positive (recurring)

**Step**: typecheck | **Exit code**: 1 | **Outcome override**: success

The import-resolution probe script generates invalid syntax when building the import list for `claude_agent_sdk`:

```
File "<string>", line 1
    from claude_agent_sdk import (,AgentDefinition,ClaudeAgentOptions,...
                                  ^
SyntaxError: invalid syntax
  FAIL: Cannot import: (
```

**Cause**: The probe script incorrectly prepends a comma to the import list, producing `import (,Symbol,...` which is syntactically invalid. This is a known recurring artifact of the typecheck script's import-list construction — it does not reflect any real problem with the SDK or skill files.

**Impact**: None — the pipeline `outcomes` object correctly marks typecheck as `success`, and verify passed with 18/18 checks.

**Suggested fix**: The typecheck script's import-list builder needs to strip leading commas before injecting into the Python probe command.
