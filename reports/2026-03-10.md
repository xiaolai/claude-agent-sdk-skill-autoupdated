- **Step**: typecheck | **Exit code**: 1
- **Outcome override**: pipeline `outcomes` records `typecheck: success` (non-blocking)
- **Last output**:
  ```
  File "<string>", line 1
      from claude_agent_sdk import (,AgentDefinition,ClaudeAgentOptions,...
                                    ^
  SyntaxError: invalid syntax
    FAIL: Some template imports don't resolve
    FAIL:   Cannot import: (

  =========================================
    Passed: 13
    Failed: 2
  =========================================

  TEMPLATE CHECK FAILED — 2 issue(s)
  ```
- **Likely cause**: The import-resolution check script constructs a synthetic `from ... import (` string that starts with a comma (`(,AgentDefinition,...`), producing invalid syntax. This is a script artifact in the check tool itself, not a problem with any SDK template. The same issue has recurred since at least 2026-03-08.
- **Suggested fix**: Fix the import-string construction in `scripts/check-versions.sh` (or whichever script builds the synthetic import) to not include a leading comma after `(`.
