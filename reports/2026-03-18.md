## Errors

### Typecheck — Exit Code 1 (recurring false-positive)

```
  File "<string>", line 1
    from claude_agent_sdk import (,AgentDefinition,ClaudeAgentOptions,...
                                  ^
SyntaxError: invalid syntax
  FAIL: Some template imports don't resolve
  FAIL:   Cannot import: (

=========================================
  Passed: 13
  Failed: 2
=========================================
TEMPLATE CHECK FAILED — 2 issue(s)
```

**Likely cause:** The import-resolution check script generates a test import line by joining symbols with commas, producing a leading comma artifact (`,AgentDefinition,...`). This is a script bug in the typecheck tool itself, not a real import failure. The pipeline outcomes correctly override this to `"success"`.

**Suggested fix:** Fix the import line generation in the typecheck script to avoid the leading comma. This has recurred across multiple daily runs.
